Reliability

Why cloud-based passes beat paper, every time

Traditional paper bus passes depend on a single physical object: lose it, damage it, or forget it, and you're stuck buying a fresh one or arguing with a conductor. A cloud-based system removes that single point of failure entirely.

  • ✓ Your pass record lives permanently in Firestore — it cannot be physically lost or torn.
  • ✓ Re-download your PDF pass unlimited times from any device, instantly.
  • ✓ Real-time sync means status changes (approval, renewal, cancellation) reflect immediately everywhere.
  • ✓ Built-in form validation and error handling catch mistakes before they become problems.
  • ✓ Firebase's distributed infrastructure means no single server outage takes the system down.

How Firebase scales with demand

⚙️ Serverless by design

Firestore and Firebase Auth auto-scale infrastructure behind the scenes — no servers to provision or manage as ridership grows.

📈 Handles traffic spikes

Festival rushes or month-start renewal surges are absorbed automatically without manual capacity planning.

Scalability

Built to grow from hundreds to millions of riders

This system uses Firestore's efficient query patterns, pagination, and indexed lookups so that performance stays snappy whether there are 100 passes or 100,000.

  • ✓ Paginated tables in the admin dashboard avoid loading entire collections at once.
  • ✓ Targeted Firestore queries (by user, status, or route) keep reads fast and cheap.
  • ✓ Real-time listeners only sync relevant data, not the whole database.
  • ✓ Firebase Hosting serves static assets via a global CDN for low latency anywhere.
